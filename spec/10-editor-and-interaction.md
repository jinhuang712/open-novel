# 10 · Editor And Interaction

本文档定义编辑器、实体高亮、命令入口、快捷键和查询入口的实现契约。它承接 design 的交互意图,但不替代视觉设计。

## 职责边界

本篇负责:

- 编辑器与作品文件的交互边界。
- 实体、概念高亮和跳转的工程语义。
- 快捷键、命令注册、查询入口和输入条焦点规则。
- 交互失败和冲突处理原则。

本篇不负责:

- 视觉 token 和具体布局。
- 知识图谱索引生成。
- Agent 输出 schema。
- 完整命令枚举。

## 主权对象

Editor and interaction 拥有:

- 编辑器适配层。
- 实体与概念高亮展示契约。
- 命令注册和快捷键冲突规则。
- 查询入口和输入条焦点语义。
- undo/redo 与 IME 安全边界。

UI 视觉和动效以 design 为准;本篇只定义实现必须遵守的交互行为。

## 输入、输出与依赖

输入是用户编辑、快捷键、命令、查询请求、知识图谱标记和 turn 状态。输出是编辑器行为、旁注、高亮、查询入口、焦点切换和交互失败提示。本篇依赖 project storage 保存作品事实,依赖 knowledge graph 提供标记,依赖 design 定义视觉呈现。

## 技术路径

编辑器显示作品正文,并从知识图谱接收实体、概念和 violation 标记。高亮用于提醒和跳转,不能直接改变作品事实。

命令系统把快捷键、命令面板、查询浮层和输入条焦点统一起来。查询入口是独立交互,输入条是与 Agent 对话的入口;二者不能混成一个隐式模式。

快捷键必须尊重文本输入、IME 组合态、浮层焦点和审批卡焦点。任何危险操作都必须进入确认或审批路径。

## 失败语义

- 高亮索引过期:展示弱化状态或不展示,不能误报为最新事实。
- 快捷键冲突:当前焦点上下文优先,不可猜测执行。
- 查询失败:展示可解释失败,不清空用户输入。
- 编辑器状态和外部文件冲突:交给项目存储层的冲突语义处理。

## 用户可见结果

用户看到的是稳定的写作纸面、必要的旁注、可靠的快捷键、独立查询入口和明确的输入焦点。系统不让工具层 UI 抢走写作主体。
