# 02 · Runtime State

本文档定义会话记忆、经验、过程历史和调试日志如何存在。读完本篇应能理解:系统记住什么、忘掉什么、什么会进入上下文、什么只用于 trace,以及 Reflector 关闭后到底发生什么。

## 要解决的问题

创作工作台需要记住两类东西:

- 作者刚刚和系统聊过什么,以便当前对话连续。
- 作者长期偏好和本书经验,以便越写越贴合。

它也需要记录第三类东西:系统执行过程,例如调用了哪个 Agent、用了哪些工具、哪里失败。这类记录只用于诊断,不能变成作品事实。

## 三库边界

| 数据库 | 主要内容 | 能否作为作品事实 |
|---|---|---|
| runtime.db | 跨项目 thread、message、压缩摘要、会话恢复轻量状态 | 否 |
| index.db | 每项目实体、概念、关系、锚点、审批和项目经验 | 部分是 |
| session_history.db | 每项目模型调用、tool run、trace、成本、debug 事件 | 否 |

这三者的边界是主权边界。过程日志再完整,也不能用来恢复章节正文;runtime message 再详细,也不能覆盖项目设定。

## 四层记忆模型

系统上下文按四层记忆组织:

| 层 | 内容 | 用途 |
|---|---|---|
| L1 工作记忆 | 当前 turn 输入、当前章节片段、临时分析结果 | 支撑本次生成 |
| L2 会话历史 | 最近消息、thread 摘要、对话压缩 | 保持对话连续 |
| L3 用户经验 | 用户采纳/否决后提炼的风格和偏好 | 注入写作习惯 |
| L4 项目事实 | 角色、世界观、关系、伏笔、章节事实 | 保证一致性 |

L4 优先级最高。用户经验不能覆盖项目事实;会话历史不能覆盖用户刚刚的显式指令;模型不能凭过程日志补事实。

## 会话记忆

会话记忆回答“这段对话之前说过什么”。它跨项目存在,但必须以 project 和 thread 隔离,避免 A 项目的上下文进入 B 项目。

会话历史可以压缩,但压缩失败不能删除原始消息。压缩摘要只用于减少上下文体积,不是原文的唯一存档。

## 过程历史

过程历史回答“系统刚刚做过什么”。它记录 turn、Agent、tool、LLM call、stream event、成本和错误。它服务 Trace 面板、调试面板、恢复判断和实施审计。

过程历史是 best-effort:写入失败不应该阻断已经完成的作品写入,但必须暴露诊断缺口。debug 面板只能只读展示过程历史,不能从过程历史恢复项目事实。

## Reflector 与经验

Reflector 在 turn 完成后,从用户明确采纳、修改、否决和反馈中提炼经验。经验是项目级长期记忆,由项目事实层持久化,由上下文装配按权重和适用范围注入。

经验管理有四个用户可见动作:

- 继续学习新经验。
- 暂停学习新经验。
- 调整已有经验的权重或适用范围。
- 删除已有经验。

Reflector 关闭后,系统不再学习新经验;已经沉淀的经验仍继续生效,直到用户在 Settings 中调低、关闭或删除。这保证“关闭学习”不会突然改变既有写作手感。

## 经验进入上下文

经验不能由 Agent 自己随意挑选。Context builder 负责根据任务、章节、Agent 类型和冲突情况挑选经验。经验与当前用户显式指令冲突时,当前指令优先;经验与项目事实冲突时,项目事实优先。

经验注入必须可追溯。用户应能看到某次生成受哪些经验影响,至少能在 Trace 或 Settings 中查到来源和权重。

## 恢复语义

恢复分为三类:

- 会话恢复:从 runtime history 恢复最近对话状态。
- turn 恢复:从 turn 状态和审批状态恢复运行结果。
- 调试恢复:从 session history 展示过程。

只有 turn 状态和项目事实能决定业务结果。流式事件或过程日志缺失时,前端应从持久 turn 状态重新判断,而不是把浏览器内存中的半截事件当真。

## 失败语义

| 失败 | 系统行为 |
|---|---|
| 会话读取失败 | 当前 turn 可少历史运行,但必须提示/记录 |
| 历史压缩失败 | 保留原始消息,延后压缩 |
| 过程日志写入失败 | 不阻断作品写入,但 Trace 标记不完整 |
| 经验写入冲突 | 合并、排队或失败提示,不能重复叠加错误经验 |
| 经验注入冲突 | 当前显式指令和项目事实优先 |
| Reflector 关闭 | 不产生新经验,已有经验按设置继续使用 |

## 用户可见结果

用户看到的是:系统是否记得上下文、是否正在学习新经验、哪些经验影响了生成、Trace 是否完整、Debug 面板是否有诊断缺口。用户不需要看到消息表或日志表字段。
