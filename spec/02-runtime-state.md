# 02 · Runtime State

本文档定义运行时记忆和过程历史的边界。它说明系统如何记住对话、压缩历史、记录过程,以及这些数据为什么不能混入项目事实。

## 职责边界

本篇负责:

- 跨项目会话记忆的主权边界。
- 单项目过程历史的主权边界。
- 用户经验、Reflector 学习结果与可见管理语义。
- 会话恢复、历史压缩和过程审计的失败语义。

本篇不负责:

- 项目事实表结构。
- Agent 输出 schema。
- 具体 UI 布局。
- 完整 message 表和日志表字段。

## 主权对象

运行时状态分为两类:

- 会话记忆:thread、message、压缩摘要和跨进程恢复所需的轻量状态。
- 过程历史:模型调用、工具运行、cascade 事件、trace、成本和调试记录。

用户经验是项目级长期记忆,但它由运行时学习链路产生,由项目事实层持久保存,由设置界面暴露管理入口。

## 输入、输出与依赖

输入是用户 turn、Agent 消息、Reflector 学习结果、过程事件和设置开关。输出是可恢复会话、压缩历史、过程 trace、经验状态和用户可管理的记忆结果。本篇依赖 project storage 持久化项目级经验,依赖 settings 暴露管理入口,并为 agent runtime 与 context builder 提供历史来源。

## 技术路径

会话记忆服务于“这段对话上下文还记得什么”。它跨项目存在,但必须通过项目身份和 thread 身份隔离,不能让一本书的上下文污染另一本书。

过程历史服务于“系统刚刚做过什么”。它用于 trace、调试、成本统计和恢复判断,不是作品事实恢复来源。

Reflector 在 turn 完成后提炼经验。关闭 Reflector 后,系统不学习新经验;已经沉淀的经验仍然可被上下文装配使用,除非用户在设置中调整或删除。

## 经验管理语义

用户必须能看到、调低、调高或删除系统学到的经验。经验管理不是隐藏调参,而是产品承诺的一部分。系统需要区分:

- 新经验是否继续产生。
- 已有经验是否继续注入。
- 用户是否主动修改经验权重。
- 某条经验是否被删除或不再参与装配。

## 失败语义

- 会话历史读取失败:当前 turn 可以显式降级为少历史或无历史,但必须记录 trace。
- 历史压缩失败:不丢原始消息,延后压缩。
- 过程日志写入失败:不应阻塞作品写入,但必须暴露诊断缺口。
- 经验写入冲突:不能重复叠加错误经验,需要合并、排队或让后续实现明确冲突策略。

## 用户可见结果

用户看到的是系统是否记得上下文、是否产生新经验、能否编辑经验、以及 trace 是否完整。用户不需要看到内部日志表,但需要能追溯一次生成为什么受某条经验影响。
