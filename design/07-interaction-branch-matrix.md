# design/07 — 核心交互分支矩阵

本篇用于验收高保真原型是否覆盖作者真正会遇到的核心分支。范围控制在 16 条,不追求穷举所有异常;每条都必须能在现有设计文档或原型中找到对应状态、入口或降级表达。

## 覆盖原则

- 只覆盖作者可感知的状态分支:启动、搜索、写作、审批、读者预演、设置和失败恢复。
- 不把内部文件名、相对路径、目录结构、存储实现、schema 或调试字段当成作者界面分支。
- 快捷键、产品固有名和设计文档编号可以保留,但主标签必须是中文。
- 原型数据可以是样例,但样例也不得向作者暴露 `.md`、`characters/`、`outline` 等实现线索。

## 验收口径

- 每个分支至少有一处 Markdown 交互契约和一处原型或原型入口可验证。
- 原型主标签使用中文;英文只允许出现在快捷键、产品固有名、设计者说明或开发构建只读说明。
- 作者界面不得展示实现路径、文件扩展名、目录层级和存储结构。
- 失败分支必须说明作者能做什么,不能只显示内部错误名。
