# design/ui-language-map — 作者界面中文化映射

本篇用于验收原型和交互文档中的作者可见文案。规则是:作者看到中文主标签和作品语义;设计者、开发者可以在文档说明中看到技术对照,但不能把它投到真实作者界面。

## 作者界面红线

- 不显示 `.md` 扩展名、相对路径、目录名或文件夹结构。
- 不显示 storage、schema、数据库、内部事件、内部字段名或工具函数名。
- 不用 Settings / Workspace / Agents / Rules / Memory / ReaderPanel / Universal Search / ApprovalCard 作为作者主标签。
- 不用「路径」作为变更来源说明;改用「位置」「章节」「角色卡」「设定卡」「来源段落」。
- 快捷键、URL、产品名 Open Novel 和少量设计固有名可保留。

## 映射表

| 原始或内部用语 | 作者界面用语 | 说明 |
|---|---|---|
| Settings | 设置 | 原型标题、按钮、命令统一用「设置」。 |
| Workspace | 作品库 / 存放位置 | 启动页说明可写「作品存放位置」,不显示 workspace。 |
| API key / API Keys | 连接凭据 | 帮助文案可解释“模型服务凭据”,主标签不用 API。 |
| Agents | AI 角色 | 七个角色用中文职能名。 |
| Rules | 创作守则 | 指五大网文守则。 |
| Memory | 记忆 / 经验 | 设置分区主标签用「记忆」。 |
| ReaderPanel | 读者预演 | 面向作者主标签用「读者预演」或「读者报告」。 |
| Universal Search | 统一搜索 | 可在设计文档中作为能力名补充,原型主标签用中文。 |
| ApprovalCard | 审批卡 | 作者看到「审批卡」「整批审批」。 |
| Cascade | 连带修改 / 影响范围 | 按语境写「整批审」「连带修改」。 |
| schema | 数据格式 / 契约 | 只出现在设计者说明或技术文档引用。 |
| spec | 技术契约 | 只出现在设计文档上游引用或设计索引说明。 |
| characters/lin.md | 林溪角色卡 | 不显示目录或扩展名。 |
| outline.md | 大纲 | 不显示扩展名和锚点短码。 |
| items/luopan.md | 青铜罗盘设定卡 | 不显示目录。 |
| path / 路径 | 位置 / 来源 | 审批卡、快速打开、搜索结果均用作品语义。 |
| quick open file | 快速打开作品内容 | 入口可以保留 Cmd+P 快捷键。 |
| @file reference | @内容引用 | 结果展示角色、章节、设定,不展示文件路径。 |

## 保留英文清单

| 类型 | 可保留示例 | 原因 |
|---|---|---|
| 快捷键 | Cmd+P, Cmd+K, Esc, Shift+Shift | 键盘固有表达。 |
| 产品名 | Open Novel | 品牌名。 |
| 设计索引说明 | design/README、prototype 编号 | 仅设计者入口,不属于作者包。 |
| 开发构建只读说明 | dev build, token | 只允许在开发构建或设计验收说明中出现。 |
| URL 或服务名 | platform.deepseek.com | 外部服务地址不可中文化。 |

## 原型验收清单

- 原型卡片标题、按钮、分区、结果行主标签均为中文。
- 原型样例数据中的作品位置用「角色卡」「章节段落」「设定卡」表达。
- 设计索引页可以说明它是设计者入口,但卡片主标题仍使用中文。
- 若 `rg` 仍命中英文或技术词,必须能归为设计文档引用、快捷键、产品名、开发说明或本映射表本身。
